# vip-ng-lab

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可能来自不同时期、不同用途的练习或演示项目，不指向任何特定域名或网站。

## 项目简介

vip-ng-lab 是一个以 HTML 页面为主要内容的归档仓库。所有页面均为独立文件，不依赖后端服务或数据库，可直接在浏览器中打开运行。适合用于保存前端实验、原型设计、静态页面等。

## 目录说明

```
/
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放主要 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 静态资源（图片、样式、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有独立的 HTML 页面文件。
- `assets/`：页面可能引用的公共资源，建议按类型分目录存放。
- 根目录下的 `index.html` 可作为导航页或默认入口（如有需要）。

## 页面归档说明

- 每个页面均为独立、自包含的 HTML 文件（除非引用 `assets/` 中的资源）。
- 页面之间不共享状态，无跨页面依赖。
- 部分页面可能包含外部 CDN 引用（如字体、图标库等），但均以公开可用库为准。
- 归档页面可能包含未完成的实验性代码或注释，请按需使用。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来修正错误、优化页面或添加新内容。
- 添加新页面时，请尽量放入 `pages/` 目录，并保持命名清晰。
- 请勿在页面中添加指向具体商业网站、诱导点击或营销性质的内容。
- 本仓库仅用于技术展示与归档，不提供任何形式的服务或担保。
